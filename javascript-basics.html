Javascript Basics

Introduction

The goal in this short intro is to get you coding. Over the course of the next few months you will learn more.

Variables

Let's start with the most basic of basics. Variables are used to store values. In Javascript variables can be of multiple types. There are 8 basic primative types.

  • Number - Used to store numeric values
  • String - Used to store text values
  • Boolean - Used to store true or false values
  • Object - Much more complex than the other types, I will explain more later.
  • undefined - You will use this a lot! The variables has no value.
  • null - It's null...it's weird. It provides unexpected results in equality, try not to use it.
  • BigInt - We won't use this
  • Symbol - You will use this, but you won't know you are using it. Don't worry about it now.

In javascript you declare a varaible with let or const. Let is how you declare a variable that you plan to change. Const is how you declare a constant variable that does not change. You should default to const until you have to use let. 


            let result = 0;
            const firstNumber = 5;
            const secondNumber = 10;
            result = firstNumber + secondNumber;
            console.log(result); // Output: 15 to the console in your web inspector in the browser.

Equality and Assignments

You will often test for equality in javascript or assign values to variables. The = operator is an assignment operator. It means take the value on the right side and put it into the variable on the left. It is not testing equality, it is assigning!

== is a test for equality. But it provides unexpected results. It will do type conversions then compare for equality. For exmaple if you compare the string "0" to the number 0, it will return true. If you compare the string "0" to the boolean false, it will return true! There are many edge cases and you should not use this.

=== is a test for equality. It will not do type conversions. It will compare the value and the type. If the types are different, it will return false. You can also do > or < as you would expect. You will notice the //. This is just a comment and will not be viewed as code. 

                
    "0" = 0 // Error, you can't assign a value to "0".
    "0" == 0 // True
    "0" == false // returns True
    "0" == true // returns false
    "0" === 0 // False!
    const age = 25;
    age === 21 // False
    age >= 21 // True

There are other control structures we will learn about in person. I will just give a few examples below. Here is the structure of an if statement. You don't have to include the else or else if. 


    const age = 25;
    if (age >= 21) then {       // First case
        console.log("They can drink.")
    } else if (age <= 12) {     // You can add another condition if the first one is false
        console.log("They can have orange juice.")
    } else {                    // Covers all other cases
        console.log("Watch out for them in this age group")
    }

Here is a for loop


    let age = 20;
    for (let i = 0; i < 10; i++) { // Start at 0, go to 9 incrementing by 1.
        age = age + 1
        console.log('They are ' + age + ' years old');
    }

The above code will print "They are 10 years old" "They are 11 years old" ... "They are 19 years old". There is much much more to go over. But let's put this into action.
